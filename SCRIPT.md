# 脚本文件说明

## 脚本文件

脚本文件为 `YAML` 格式, 用于指导程序执行相关操作。

### 脚本文件基础结构

```yaml
# 脚本名称
name: 标记/锁定双爆圣遗物
# 配置描述
description: 标记/锁定双爆圣遗物
# 标签 [可选]
tags:
  - 双爆
# 执行模式 [可选值: 处理当前圣遗物 处理所有圣遗物 强化当前圣遗物 摧毁圣遗物 分解圣遗物 卸下圣遗物 快速装备圣遗物 编辑快速装备方案]
mode: 处理当前圣遗物

# 各个模式独有配置
# ...
```

配置文件支持的模式: [处理当前圣遗物](#模式-处理当前圣遗物), [处理所有圣遗物](#模式-处理所有圣遗物), [强化当前圣遗物](#模式-强化当前圣遗物), [摧毁圣遗物](#模式-摧毁圣遗物), [分解圣遗物](#模式-分解圣遗物), [卸下圣遗物](#模式-卸下圣遗物), [快速装备圣遗物](#模式-快速装备圣遗物), [编辑快速装备方案](#模式-编辑快速装备方案)。

### 模式: 处理当前圣遗物

> [!Important]
> - `规则列表` 按顺序执行, 上一条的 `动作` 会影响下一条 `筛选表达式` 的判断。
> - `筛选表达式` 推荐添加 `星级` 约束, 避免操作了有纪念意义的非5星圣遗物。
> - 使用 `强化` 功能前, 请先在单一圣遗物上测试, 避免造成不必要的损失。

模式说明: 此模式下, 程序只处理当前游戏内背包-圣遗物界面正在浏览的圣遗物。

此模式下, 需要添加 `筛选器(filter)` 和 `规则列表(rules)` 配置。`筛选器` 用于设置游戏内的圣遗物筛选条件, `规则列表` 则用于指导程序执行相关操作。

表达式语法 `筛选表达式`/`强化表达式` 请参考: [表达式语法](#表达式语法)。

```yaml
mode: 处理当前圣遗物

# 表达式数字小数精度 [默认值: 2] [可选范围: 1-4]
expression_precision: 2
# 严格模式 (严格模式下, 圣遗物所需的属性全部识别正确才会执行动作) [默认值: 禁用] [可选值: 启用 禁用]
strict_mode: 禁用
# 初始化强化设置 (开启阶段强化, 关闭允许添加五星材料) [默认值: 禁用] [可选值: 启用 禁用]
ignore_enhance_error: 禁用
# 忽略强化异常 (异常: 材料不足, 摩拉不足) [默认值: 启用] [可选值: 启用 禁用]
init_enhance_settings: 启用

# 筛选器[可选]
# 简单筛选 [可选值: 重置 开启按时间顺序展示 关闭按时间顺序展示]
filter: 重置
# 详细筛选
filter:
  # 套装名称[可选]
  sets:
    - 纺月的夜歌
    - 穹境示现之夜
  # 锁定状态[可选] [可选值: 标记 仅锁定 未锁定]
  lock_states:
    - 标记
    - 仅锁定
    - 未锁定
  # 等级状态[可选] [可选值: 满级 未满级]
  level_state: 未满级
  # 装备状态[可选] [可选值: 已装备 未装备]
  equipment_state: 未装备
  # 来源[可选] [可选值: 祝圣之霜定义]
  source: 祝圣之霜定义

# 规则列表
rules:
    # 规则描述
  - description: 标记/锁定双爆圣遗物
    # 筛选表达式
    expression: |-
      星级 == 5
    # 执行动作 [可选值: 点击锁定 点击标记 仅锁定 标记 取消标记 锁定 强化]
    action: 强化
    # 强化表达式 (表达式真值时结束强化) [强化动作必填]
    enhance_expression: |-
      等级 == 16
```

### 模式: 处理所有圣遗物

模式说明: 此模式下, 程序会处理游戏内背包-圣遗物界面所有的圣遗物。

此模式的配置与 `处理当前圣遗物` 模式相同, 以下是一些独有的配置项。

```yaml
mode: 处理所有圣遗物

# 表达式数字小数精度 [默认值: 2] [可选范围: 1-4]
expression_precision: 2
# 严格模式 (严格模式下, 圣遗物所需的属性全部识别正确才会执行动作) [默认值: 禁用] [可选值: 启用 禁用]
strict_mode: 禁用
# 初始化强化设置 (开启阶段强化, 关闭允许添加五星材料) [默认值: 禁用] [可选值: 启用 禁用]
ignore_enhance_error: 禁用
# 忽略强化异常 (异常: 材料不足, 摩拉不足) [默认值: 启用] [可选值: 启用 禁用]
init_enhance_settings: 启用

# 停止星级 (扫描到此星级的圣遗物时停止后续操作) [可选] [可选范围: 1-5]
stop_star: 1

# 其他相同配置参考模式: 处理当前圣遗物
# ...
```

### 模式: 强化当前圣遗物

模式说明: 此模式下, 程序会强化游戏内背包-圣遗物界面正在浏览的圣遗物。

此模式下, 需要添加 `强化表达式` 配置。`强化表达式` 用于指导程序执行强化操作。

```yaml
mode: 强化当前圣遗物

# 表达式数字小数精度 [默认值: 2] [可选范围: 1-4]
expression_precision: 2
# 严格模式 (严格模式下, 圣遗物所需的属性全部识别正确才会执行动作) [默认值: 禁用] [可选值: 启用 禁用]
strict_mode: 禁用
# 初始化强化设置 (开启阶段强化, 关闭允许添加五星材料) [默认值: 禁用] [可选值: 启用 禁用]
ignore_enhance_error: 禁用
# 忽略强化异常 (异常: 材料不足, 摩拉不足) [默认值: 启用] [可选值: 启用 禁用]
init_enhance_settings: 启用

# 强化表达式 (表达式真值时结束强化)
expression: |-
  等级 == 16
```

### 模式: 摧毁圣遗物

模式说明: 此模式下, 程序会摧毁背包中指定等级的圣遗物，圣遗物摧毁换摩拉。

```yaml
mode: 摧毁圣遗物

# 需要摧毁的等级 [可选范围: 1-4]
destroy_stars:
  - 1
  - 2
  - 3
  - 4
```

### 模式: 分解圣遗物

模式说明: 此模式下, 程序会分解背包中指定等级的圣遗物，圣遗物分解换材料。

```yaml
mode: 分解圣遗物

# 需要分解的等级 [可选范围: 1-4]
decompose_stars:
  - 1
  - 2
  - 3
  - 4
```

### 模式: 卸下圣遗物

模式说明: 此模式下, 程序会卸下全部角色身上的圣遗物。

```yaml
mode: 卸下圣遗物

# 需要卸下圣遗物的角色 [可选: 未填写则表示所有角色]
characters:
  - 白术
  - 七七
```

### 模式: 快速装备圣遗物

模式说明: 此模式下, 程序会为角色快速装备圣遗物 (利用游戏内默认快速装备功能实现)。

```yaml
mode: 快速装备圣遗物

use_main_affix: 禁用 # 套装无法满足时是否启用满足主属性的推荐方案 (禁用: 套装不满足时不装备) [默认值: 禁用] [可选: 启用 禁用]

# 需要装备圣遗物的角色, 角色越靠前越先装备圣遗物
characters:
  - 玛薇卡
  # 使用推荐方案
  - 哥伦比娅
  # 使用自定义方案
  - name: 哥伦比娅
    plan: 方案二 # [可选: 方案一 方案二]
    use_main_affix: 禁用 # 套装无法满足时是否启用满足主属性的推荐方案 (优先级高于全局设置 `use_main_affix`, 若未配置则以全局配置为准) [默认值: 禁用] [可选: 启用 禁用]
```

### 模式: 编辑快速装备方案

模式说明: 此模式下, 程序会编辑角色的快速装备方案。

```yaml
mode: 编辑快速装备方案

overwrite: 启用 # 是否启用覆盖模式 (启用: 覆盖原有方案) [默认值: 禁用]

characters:
  - name: 哥伦比娅 # 角色名称
    plan_one: # 方案一
    # pan_two: # 方案二
      overwrite: 启用 # 是否启用覆盖模式 (优先级高于全局设置 `overwrite`, 若未配置则以全局配置为准) [可选: 启用 禁用]
      sets: # 套装 (1~2个) [可选: 圣遗物套装名称]
        - 晨星与月的晓歌
      main_affix: # 主属性
        sands: 生命值百分比 # 时之沙
        goblet: 生命值百分比 # 空之杯
        circlet: 暴击伤害 # 理之冠
      priority_minor_affixes: # 优先级副属性 (1~3个) [可选: 副属性名称]
        - 暴击率
        - 暴击伤害
        - 元素精通
      secondary_minor_affixes: # 次级追加副属性 (1~2个) [可选: 副属性名称]
        - 生命值百分比
        - 元素充能效率
      other_option: 仅未装备 # 圣遗物来源 [可选: 全部 仅未装备] [默认值: 仅未装备]
```

## 表达式语法

表达式由逻辑运算符、算术运算符、条件运算符、算子、括号、注释等组成。

### 支持操作

- 算子: 数字 布尔
- 算术运算符: + - * /
- 逻辑运算符: && || !
- 条件运算符: > < >= <= == !=
- 优先级运算符: ( )
- 注释: #

### 数字算子

- 整数: 123
- 浮点数: 3.14
- 圣遗物等级: 等级
- 圣遗物星级: 星级
- 圣遗物词条值:
  - 主词条: 主:生命值
  - 副词条: 生命值
- 副词条个数: 副词条个数

### 布尔算子

- 真: 真
- 假: 假
- 圣遗物名称: 战狂的翎羽
- 圣遗物部位: 死之羽
- 圣遗物套装名: 战狂
- 圣遗物词条名:
  - 主词条: 主:生命值
  - 副词条: 生命值
- 已锁定: 已锁定
- 已标记: 已标记
- 已装备: 已装备
- 祝圣之霜定义: 祝圣之霜定义

### 注释

`#` 符号后到行尾的内容视为注释, 表达式解释器会忽略注释内容。

```txt
# 前注释
真 # 中间注释
# 后注释
```

### 示例

- 带有双爆词条的圣遗物

```txt
(暴击率 && 暴击伤害)
|| (主:暴击率 && 暴击伤害)
|| (暴击率 && 主:暴击伤害)
```

- 未锁定的战狂羽毛

```txt
!已锁定 && 战狂 && 死之羽
```
